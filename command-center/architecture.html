The 6-Layer Stack
1
Data Ingestion , Public Signal Sources
We pull from 68+ public data sources every weekday. Permit filings, EDGAR 10-K filings, BLM mining claims, utility interconnection queues, WSDOT/Caltrans infrastructure projects, zoning variances, pre-bid attendance rosters, federal earmarks. Every row becomes a signal candidate.

2
AI Normalization + Scoring
Python pipeline normalizes every signal into a canonical schema. Claude API resolves ambiguous company domains. A scoring agent assigns delay_pressure, constraint_score, and probability scores. Residential noise is filtered. Only commercial/industrial permits with crane potential pass through.

3
Contact Enrichment
From permit address → contractor company name → domain guess → contact page scrape → email pattern generation → MillionVerifier. Currently 367 companies with names, 74 verified emails from 50-company sample. Google CSE people search (blocked) would add person names + titles.

4
Predictive Analytics , Catch-Up Radar
GitHub Actions runs twice daily. Signals are ranked by delay_pressure + constraint_score + weather enrichment + inspection backlog. Top projects go to Google Sheets (3 tabs: Merged, EarlySignals, Call_List) and a Slack digest to #catch-up-radar. This is the daily intel product.
